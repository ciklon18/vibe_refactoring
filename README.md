# DevOps/SRE Docker Practice

Практика по контейнеризации небольших веб-сервисов.

## Правила практики

1. Клонируем репозиторий
2. Выбираем директорию с проектом
3. Добиваемся локального запуска по инструкции
4. Создаем `Dockerfile` для контейнеризации веб-приложения
5. Создаем `docker-compose.yml` для полного запуска сервиса

## Быстрый старт через Docker Compose для Spring-приложения

В репозитории есть `docker-compose.yml`, который поднимает Spring Boot приложение вместе с PostgreSQL.

1. Скопируйте переменные окружения и при необходимости отредактируйте значения:

```sh
cp .env.example .env
```

2. Запустите сервисы:

```sh
docker compose up --build
```

Проброшенные порты по умолчанию:

- Spring Boot: `8081` (контейнерный порт приложения `8080`)
- PostgreSQL: `5434` (контейнерный порт БД `5432`)

Для изменения портов или учетных данных обновите `.env` и перезапустите `docker compose`.
